## 어떤 모드인가요?

벽 하나를 사이에 두거나, 방에서 밖으로 나가거나, 물속으로 들어가면 소리가 달라집니다. 현재 월드 구조를 계속 확인하기 때문에 블록을 설치하거나 부숴도 재생 중인 소리에 반영됩니다.

## 주요 기능

- 벽의 재질과 두께에 따른 차폐
- 모서리, 출구, 이어진 구조물을 따라 전달되는 소리
- 벽, 바닥, 천장에서 돌아오는 반사음
- 공간 크기, 출구, 표면 재질에 따라 달라지는 잔향
- 공기, 물, 용암 경계에서 달라지는 소리
- 움직이는 음원과 플레이어, 변경된 블록의 지속적인 반영
- 네 가지 품질 프리셋과 세부 설정
- JSON으로 설정하는 블록 및 유체별 음향 특성

## 동작 방식

월드에서 발생하는 모든 위치 기반 소리는 같은 전파 파이프라인을 거칩니다. 상황별로 미리 만들어 둔 방 프리셋을 고르는 방식은 아닙니다.

```text
월드 형상과 재질
        ↓
직접음과 재질 투과
        ↓
회절과 구조 전달 경로
        ↓
초기 반사음과 후기 잔향 추적
        ↓
거리, 대기, 매질에 의한 손실
        ↓
실시간 소프트웨어 믹서
```

### 직접음과 차폐

음원과 귀 사이에 중심 광선 하나만 쏘지 않고, 첫 번째 프레넬 영역 크기의 유한한 파면을 여러 지점에서 샘플링합니다. 각 경로는 블록의 실제 충돌 모양과 교차시키고, 재질 내부를 지나간 길이를 모두 누적합니다. 그 길이와 재질별 투과 손실을 8개 주파수 대역에 따로 적용하기 때문에 벽이 여러 겹이면 손실도 계속 쌓이며, 얇거나 일부만 막힌 블록이 무조건 1m짜리 벽처럼 처리되지 않습니다.

### 회절

직접 파면이 막히면 청크, 출구(포털), 서로 보이는 구간, 모서리 노드로 이루어진 그래프를 필요한 범위에서 만듭니다. A*가 문과 모서리, 여러 번 꺾이는 통로를 잇는 전체 경로를 찾은 뒤, 각 경로에 프레넬 파라미터를 이용한 knife-edge 회절과 Fresnel–Kirchhoff 감쇠 근사를 주파수 대역별로 적용합니다. 여러 출구로 도달한 소리는 한 출구만 고르지 않고 에너지와 도착 방향을 함께 합칩니다.

### 거리 감쇠와 공기 흡음

빈 공간의 거리 감쇠는 구면파 확산을 따릅니다. 음압은 대략 `1 / 거리`로 줄어들고, 음의 세기는 거리의 제곱에 반비례하므로 거리가 두 배가 될 때마다 약 6 dB씩 작아집니다.

공기 흡음은 단순한 저역 통과 필터가 아니라 ISO 9613-1 계산을 8개 주파수 대역에 각각 적용합니다. 온도, 상대 습도, 기압으로 산소와 질소의 분자 이완 손실을 계산하고, 거리 `d`를 이동한 각 대역에 `exp(-αd)`만큼의 음압 감쇠를 적용합니다. 그래서 임의의 청취 거리 제한 없이 먼 거리일수록 고주파부터 자연스럽게 줄어듭니다.

### 반사음

주변 표면을 기준으로 image-source 방식의 1차 반사 후보를 만들고, 현재 월드 형상에서 실제로 왕복 경로가 열려 있는지 다시 검사합니다. 확인된 경로에 거리와 지연 시간, 도착 방향, 재질별 흡음·투과·산란을 적용합니다. 따라서 돌, 나무, 양털 같은 재질은 반사되는 음색과 에너지가 서로 다릅니다.

### 공간 분석과 잔향

후기 잔향은 여러 번 튕기는 몬테카를로 광선 추적으로 측정합니다. 광선은 재질에 부딪힐 때마다 에너지를 잃고 실제 출구로 빠져나가며, 출구 손실에는 Sabine 계열의 공간 에너지 모델을 사용합니다. 도착 에너지는 0.5ms 간격의 히스토그램에 모은 뒤, 뒤에서부터 누적하는 Schroeder 에너지 적분과 감쇠 곡선 회귀로 각 주파수 대역의 RT60을 구합니다.

측정된 잔향은 24kHz로 동작하는 8개 지연선의 FDN(Feedback Delay Network)으로 재생합니다. 서로 배수가 되지 않는 지연 길이, 에너지를 보존하는 정규화 Hadamard 피드백 행렬, 네 단계의 Schroeder all-pass 확산기를 사용해서 한 번의 에코가 반복되는 대신 촘촘한 잔향 꼬리가 만들어집니다. 왕복 지연 시간이 `τ`인 지연선의 피드백 이득은 `g = 10^(-3τ / RT60)`으로 계산하여 측정된 RT60 동안 에너지가 60dB 감소하도록 맞춥니다. 피드백 신호는 250Hz와 4kHz의 one-pole 필터로 저역·중역·고역을 나누고, 추적한 RT60 스펙트럼에 따라 각 대역의 감쇠율을 따로 계산합니다.

### 유체와 매질 경계

공기, 물, 용암은 각각 다른 음속과 음향 임피던스를 가진 매질로 처리합니다. 경로를 실제 매질 구간으로 나누고, 경계의 반사와 투과 에너지는 두 매질의 임피던스 관계로 계산하며, 지연 시간은 각 구간의 거리와 음속으로 계산합니다. 유체 표면에 가까운 음원은 복셀 여섯 면을 여러 패치로 나눠 결합량을 적분하므로, 잠깐 공기가 생기거나 일부만 잠겼다는 이유로 소리가 완전한 지상음 또는 수중음으로 갑자기 바뀌지 않도록 했습니다.

### 구조를 타고 전달되는 소리

블록과 접촉한 소리는 이어진 고체 구조를 진동 경로로 사용할 수 있습니다. A*가 면으로 연결된 고체 블록을 따라가며, 각 재질의 결합량과 주파수별 감쇠로 귀까지 도달하는 에너지를 계산합니다. 윗방의 발소리가 바닥을 타고 전달되는 경우도 특정 사운드 ID를 예외 처리하지 않고 같은 계산으로 처리합니다.

### 동적 갱신과 성능

월드 데이터는 변경 불가능한 청크 섹션 스냅샷으로 보관합니다. 블록이나 유체가 바뀌면 영향을 받은 섹션만 교체하고, 전파 계산은 렌더 스레드 밖에서 처리합니다. 재생 중인 소리는 음원, 귀, 월드 형상이 바뀌는 동안 다시 계산하며 오디오 콜백은 최신 방향과 음향 계수를 계속 반영합니다. 품질 프리셋은 다른 방식으로 폴백하는 것이 아니라 같은 물리 모델에서 추적량만 조절합니다.

재질별 흡음, 투과 손실, 산란, 매질 임피던스와 주파수 대역 값은 `src/client/resources/assets/acousticsystem/acoustic_materials/default.json`에 정의되어 있습니다.

## 설치

1. Minecraft 버전과 모드 로더에 맞는 JAR 파일을 준비합니다.
2. JAR 파일을 Minecraft의 `mods` 폴더에 넣습니다.
3. 게임의 **음악 및 소리 설정 → 사운드 물리학**에서 프리셋이나 세부 항목을 조절합니다.

## 빌드

현재 Fabric 대상을 빌드합니다.

```bash
./gradlew build
```

지원하는 모든 버전과 로더를 한 번에 빌드합니다.

```bash
./gradlew buildAllMinecraftVersions
```

완성된 파일은 `build/libs/<Minecraft 버전>/` 폴더에 저장됩니다.
